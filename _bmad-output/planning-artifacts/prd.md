# Product Requirements Document - project-manager

**Author:** Sue
**Date:** 2026-03-16

## Executive Summary

为小型室内设计工作室打造一款**轻便但严格**的项目管理与财务一体化系统。帮助设计工作室主理人无需过多操作就能管清楚项目、记清楚账，告别漏记错记费用的困扰。

### 核心用户痛点

设计工作室需要花大量精力管理项目和财务，日常容易漏记或错记费用明细，缺乏系统化的管理工具。

### 产品价值主张

**"不用做多余的事，也不会出错"** - 以最简洁的方式完成项目进度跟踪、客户管理、费用与收款管理、报表统计等核心功能。

### 差异化特点

- **轻便**：随时随地用简单方式维护项目数据
- **严格**：项目管理与财务管理是严肃的事情，系统设计确保数据准确无误

## Project Classification

- **产品类型**：移动应用（uniapp）
- **领域**：设计工作室项目管理与财务一体化
- **复杂度**：中等
- **项目背景**：绿地项目（新系统）

## Success Criteria

### User Success

- 用户能够通过简单操作完成项目信息和费用记录
- 用户无需额外步骤即可查看项目进度和财务状况
- 用户不会出现"输入后发现不对，需要大量修改"的情况

### Business Success

- 系统上线即可使用，无需长时间调试
- MVP版本能够在1个月内完成并投入使用

### Technical Success

- **数据准确性优先**：所有财务数据（费用、收款）必须准确无误
- **系统稳定性**：核心功能稳定运行，不出现数据丢失或错误
- **易用性**：在保证准确性的前提下，尽量简化操作流程

### Measurable Outcomes

- 项目信息录入后无需二次修改
- 费用记录与收款记录准确对应
- 系统响应流畅，操作步骤少于3步完成核心操作

## Product Scope

### MVP - Minimum Viable Product

- 客户管理（基础数据）
- 供应商管理（基础数据）
- 项目管理（7阶段进度跟踪）
- 费用管理（预算→支出→核销）
- 收款管理（定金/中期款/尾款）
- 基础报表统计

### Growth Features (Post-MVP)

- 报表统计增强（利润分析、现金流趋势）
- 合同管理基础功能
- 发票管理

### Vision (Future)

- AI合同功能（生成、审核）
- 自定义存储接口
- 高级权限管理

## User Journeys

### 用户类型

1. **设计工作室主理人** - 主要用户，全面使用系统
2. **项目经理** - 可能涉及人工费用等录入
3. **财务人员（兼职）** - 不直接进入系统，通过报表获取信息

### 旅程1：设计工作室主理人 - 核心使用场景

**开场：** 每天早上到公司，想了解昨天项目的进展和费用情况

**发展：**
- 打开手机，快速查看各个项目的进度状态
- 收到一条银行流水推送，需要核销一笔材料采购费用
- 系统自动匹配到对应项目，点击确认即可完成核销
- 查看本月应收款和已收款对比，确认资金流

**高潮：** 通过简单操作完成所有日常管理，无需额外步骤

**结局：** 清楚了解项目状态和财务状况，无需花大量时间核对

### 旅程2：项目经理 - 费用录入场景

**开场：** 项目进行中，需要录入一笔人工费用

**发展：**
- 打开系统，选择对应项目
- 选择费用类型（人工/外包），填写金额和说明
- 系统自动关联到项目的费用预算

**高潮：** 快速完成费用录入，无需复杂操作

**结局：** 费用记录准确，主理人可随时查看

### 旅程3：移动端费用核销

**开场：** 收到银行流水通知，有一笔支出需要核销

**发展：**
- 打开手机，查看待核销的银行流水
- 系统自动匹配可能的费用记录
- 确认匹配结果，完成核销

**结局：** 费用数据与银行流水准确对应，无需手动核对

### Journey Requirements Summary

- 移动端费用快速录入
- 银行流水自动匹配与核销
- 项目进度状态快速查看
- 应收/已收财务报表查看
- 费用预算关联
- 管理员用户管理

## Innovation & Novel Patterns

### Detected Innovation Areas

**核心创新：快捷且严谨的交互设计**

1. **智能默认值 + 确认机制**
   - 根据历史记录自动填充常用项目、客户、供应商
   - 费用类别智能推荐（上次选择的历史类别）
   - 金额自动格式化（如输入"1000"自动显示"1,000"）
   - 关键操作二次确认
   - 异常数据提醒（如费用远超历史平均）
   - 提交前摘要展示

2. **多样化的录入方式**
   - 手动录入
   - 截图识别（OCR）
   - 语音录入
   - 银行流水导入（自动匹配核销）
   - Excel/CSV批量导入

3. **快捷与严谨的平衡**
   - 输入步骤少于3步完成核心操作
   - 每一步都有数据校验
   - 错误可快速撤回和修改

### Validation Approach

- MVP阶段：以核心录入场景验证智能默认值准确率
- 用户反馈迭代优化默认值算法
- 逐步增加多样化录入方式

### Risk Mitigation

- 智能匹配可能存在误差 - 提供人工确认环节
- 多样化录入可能增加复杂度 - MVP优先实现核心方式

## PC Web Management Platform

### 平台定位

PC Web 端作为管理后台，提供微信小程序中无法便利处理的复杂管理操作。主要用户为设计工作室主理人和财务人员。设计原则：**完整性 > 速度**，支持复杂搜索、批量操作、数据审计。

MVP 阶段目标：建立完整的 PC 端架子（认证、权限架子、数据访问），确保 Phase 2 可直接扩展功能而无需重构。

### PC 端功能范围（按模块）

标注说明：**[MVP]** = 一期必须实现；**[Phase 2]** = 二期扩展

**客户管理：**
- [MVP] 客户列表查看（支持按名称/联系人搜索）
- [MVP] 单条客户 CRUD 操作
- [MVP] 客户分析数据查看（项目数、总金额）
- [Phase 2] 批量导入/导出（Excel/CSV）
- [Phase 2] 重复客户检测与合并

**供应商管理：**
- [MVP] 供应商列表查看与单条 CRUD
- [MVP] 支付记录查看
- [Phase 2] 支付记录按日期/金额筛选
- [Phase 2] 供应商评级/备注

**项目管理：**
- [MVP] 项目列表查看（支持按客户/状态筛选）
- [MVP] 项目详情查看与编辑（地址/预算/描述等完整字段）
- [MVP] 项目创建/删除
- [MVP] 项目归档管理（查看/恢复归档项目）
- [Phase 2] 批量操作（状态切换/归档/删除）
- [Phase 2] 高级搜索/筛选（多条件组合）
- [Phase 2] 项目数据导出

**费用管理：**
- [MVP] 费用列表查看与单条 CRUD
- [MVP] 核销记录查看
- [MVP] 费用删除后重新录入（数据修正方式）
- [Phase 2] 费用批量导入（Excel/CSV）
- [Phase 2] 费用历史追溯（操作人/操作时间/修改记录）
- [Phase 2] 核销记录修正（撤销后重新核销）
- [Phase 2] 费用审核流程（查看/修正/批准待审核费用）

**收款管理：**
- [MVP] 收款记录列表查看与单条 CRUD
- [MVP] 逾期应收提醒查看
- [Phase 2] 收款计划配置（定义分阶段收款比例和金额）
- [Phase 2] 收款对账（将实际收款与计划对应）
- [Phase 2] 收款计划调整

**报表统计：**
- [MVP] 基础报表查看（应收应付、实收实付）
- [MVP] 数据导出 Excel（简单表格格式）
- [Phase 2] 高级报表（按项目/客户/时间段分组）
- [Phase 2] 钻取分析（报表数据逐级展开明细）
- [Phase 2] 时间段对比（同比/环比）
- [Phase 2] 利润分析、现金流趋势
- [Phase 2] 自定义报表字段、PDF 导出

**系统功能：**
- [MVP] 用户账号管理（创建/编辑/删除/禁用用户）
- [MVP] 权限角色架子（数据模型预留，MVP 所有用户默认全部权限）
- [MVP] 系统设置（费用类别配置、应收阶段定义）
- [MVP] 操作日志记录（后台静默记录，MVP 不对外展示）
- [Phase 2] 角色权限细粒度配置（定义各角色可访问的功能和数据范围）
- [Phase 2] 数据审计日志查看界面
- [Phase 2] 数据备份（手动触发或定时备份）

### PC 端不提供的功能

- 不支持移动设备交互优化（仅桌面浏览器）
- 不支持实时推送通知（仅微信小程序）
- 不支持相机/相册操作（仅微信小程序）

### PC 端认证机制

- [MVP] PC 端用户账号与小程序账号统一（同一套用户体系）
- [MVP] PC 端采用用户名/密码登录（不依赖微信环境）
- [Phase 2] 支持小程序扫码登录 PC 端

### PC 端特有的业务流程

**数据管理（MVP）：** 主理人可在 PC 端对各模块数据进行完整的查看和编辑，补充小程序端不便处理的复杂操作。

**费用审核流程（Phase 2）：** 财务人员可在 PC 端批量查看待审核费用，进行修正、拒绝或批准。

**数据对账（Phase 2）：** 支持银行流水与费用的对账，允许撤销已核销记录后重新核销。

**权限管理（Phase 2）：** 管理员可配置各用户角色的功能权限和数据访问范围，小程序端同步遵守权限规则。

**数据审计（Phase 2）：** 查看所有数据修改记录，追溯操作人、操作时间和操作内容。

## Project Scoping & Phased Development

### MVP Strategy & Philosophy

**MVP Approach:** 体验MVP - 注重快捷且严谨的交互体验，让用户能够快速上手并感受到数据准确性
**Resource Requirements:** 小型开发团队（1-2人），熟悉uniapp开发

### MVP Feature Set (Phase 1)

**Core User Journeys Supported:**
- 客户/供应商基础数据管理
- 项目创建与7阶段进度跟踪
- 费用预算、支出与核销
- 收款记录（定金/中期款/尾款）
- 基础报表统计

**Must-Have Capabilities - 微信小程序端：**
- 客户管理（CRUD）
- 供应商管理（CRUD）
- 项目管理（7阶段：立项→需求确认→施工→验收→结算→维保→归档）
- 费用管理（有计划/无计划、预算→支出→核销）
- 收款管理（定金/中期款/尾款）
- 基础报表（应收应付、实收实付）

**Must-Have Capabilities - PC Web 管理后台（MVP 架子）：**
- 完整的 PC Web 应用架子（认证、路由、权限数据模型）
- 用户名/密码登录（与小程序同一用户体系）
- 各模块数据列表查看和单条 CRUD
- 基础报表查看和 Excel 导出
- 用户账号管理（创建/编辑/禁用）
- 系统设置（费用类别、应收阶段定义）
- 操作日志静默记录（后台记录，不展示界面）

### Post-MVP Features

**Phase 2 (Growth):**
- PC 端高级功能（批量操作、高级搜索、复杂报表、数据审计日志界面）
- PC 端费用审核流程和数据对账
- PC 端角色权限细粒度配置（小程序端同步遵守）
- PC 端小程序扫码登录
- 报表增强（利润分析、现金流趋势）
- 合同管理基础功能
- 发票管理

**Phase 3 (Expansion):**
- AI合同功能（生成、审核）
- 自定义存储接口
- 高级权限管理

### Risk Mitigation Strategy

**Technical Risks:** uni-cloud免费版有限制 - MVP阶段通过优化数据结构解决
**Market Risks:** 单一用户验证 - 快速迭代响应反馈
**Resource Risks:** 小型团队 - MVP精简功能，确保核心流程完整

- FR29: 用户可以查看逾期未收进度统计
- FR30: 用户可以导出Excel报表

### 7. 系统功能

- FR31: 管理员可以管理用户账号
- FR32: 用户可以设置个人权限（功能级别）
- FR33: 系统使用微信消息进行推送通知
- FR34: 用户可以上传合同文件（可选）

## Non-Functional Requirements

### Performance

- **响应时间：** 核心操作（费用录入、项目创建）响应时间少于2秒
- **启动速度：** 微信小程序启动时间越快越好
- **操作步骤：** 核心操作步骤少于3步

### Security

- **数据保护：** 所有财务数据需加密存储和传输
- **用户权限：** 根据用户角色控制数据访问权限
- **数据准确性：** 财务数据必须准确无误，支持数据校验

### Integration

- **银行流水：** 支持导入银行流水进行费用核销
- **微信通知：** 使用微信消息进行推送通知
- **报表导出：** 支持导出Excel报表
